Создание маршрутов. Заказы.

Важно!

На этапе тестирования при нажатии кнопки "Получить" загружаются ВСЕ заказы выбранного дня.

При эксплуатации в обычном режиме загружаются только те заказы, которые не внесены в маршрутные листы.

Общий вид таблицы и доступные колонки.

Для удобства обработки данных можно использовать фильтр заказов 

Описание реквизитов фильтра:

"Клиент" - отбор заказов по строке наименования клиента (регистр значения не имеет, поиск происходит по вхождению строки поиска в любом месте).

"Точка" - отбор заказов по строке наименования точки клиента (регистр значения не имеет, поиск происходит по вхождению строки поиска в любом месте).

"Адрес" - отбор заказов по строке адреса заказа. Если в заказе выбрана точка клиента, то по адресу точки, иначе - по адресу клиента (регистр значения не имеет, поиск происходит по вхождению строки поиска в любом месте).

"Город" - отбор заказов по строке населенного пункта заказа. Аналогично поиску по адресу.

"Группа клиентов" - отбор заказов по группе справочника "Контрагенты".

"Продукция" - отбор заказов по одному из видов продукции.

"Отмеченные" - отбор заказов с установленным флагом участия в расчете.

"Не отмеченные" - отбор заказов с не установленным флагом участия в расчете.

"Не" - инвертирует отбор соответствующего поля. Применяется только если в основном поле отбора установлено значение.

Кнопки:

"Уст. фильтр" - применить фильтр к таблице заказов. После применения фильтра в таблице будут отображены только те заказы, которые удовлетворяют условиям фильтра. Фильтр будет применен по принципу сложения фильтров по каждому заполненному полю отбора.

"Сбросить поля" - установить значения реквизитов фильтра по умолчанию. Нажатие кнопки не приводит к применению или сбросу фильтра.

"Уд. фильтр" - отменить действие фильтра к таблице заказов. После нажатия кнопки будут отображены все загруженные из учетной системы заказы.
