Основные понятия. Штрафы.

При проведении расчета программе необходимо дать возможность отбросить некоторые заказы. 

Отбрасывание заказов происходит в следующих случаях:

  • включение в маршрут данного заказа приводит к значительному увеличению длины маршрута. Другими словами ради одного клиента машине придется проехать значительное расстояние и отбрасывание этого заказа сделает маршрут значительно менее дорогим. Увеличением значения штрафа можно принудить программу включить все такие заказы в маршрут.
  • суммарное количество груза в распределямых заказах превышает суммарную вместимость машин. Можно только увеличить значение штрафа некоторых заказов, что бы вместо одних были отброшены другие. 
  • невозможно исполнить все заказы с соблюдением окон времени доставки и времени работы машины. Можно только увеличить значение штрафа некоторых заказов, что бы вместо одних были отброшены другие. 
  • нет выбранных для расчета машин, в список допустимых групп клиентов которых входит группа клиента заказа (например выбраны для расчета только Исузу, у которых установлены группы "Оптовики" и "Предприятия", и пытаемся на них распределить заказ клиента из группы "Частные лица-адреса"). Можно только скорректировать список допустимых групп машины.
  • нет выбранных для расчета машин, которые являются допустимыми для данного заказа (например выбраны для расчета Исузу, а пытаемся распределить заказ в котором в списке допустимых указана только Таврия). Можно скорректировать список допустимых машин отброшенных заказов  - добавить машину из числа выбранных для расчета или очистить список допустимых машин заказа. Или добавить в расчет требуемую машину.
  • суммарное количество товара заказов дополняемого маршрута превышает установленную вместимость машины по данному виду товара (например вместимость машины 90 баллонов, а маршруте суммарное количество баллонов 91). Один или несколько заказов будут выброшены из маршрута. Можно увеличить вместимость данной машины по этому виду товара, уменьшить количество товара в заказе.
  • заказ содержит товар, по которому в выбранных для расчета машинах не указана вместимость (например заказы с водой пытаемся распределить на Таврии). Аналогично предыдущему пункту.

Для указания приоритетов заказов с точки зрения возможности их отбрасывания используются штрафы.

Штраф делает более невыгодным ("дорогим") маршрут, если данный заказ будет отброшен. Упрощенно говоря, если штраф равен 30 минутам, то отбрасывание такого заказа делает маршрут "длинее" на 30 минут.

Основные параметры расчета штрафов за непосещение устанавливаются в учетной системе. 

Базовый штраф, мин.  - минимальная величина штрафа. Для заказов в режиме "Оптимизация" применяется только это значение.

Штраф за час уменьшения окна, мин.  - прогрессивное увеличение штрафа на указанную величину кратно количеству часов разницы между продолжительностью стандартного окна времени доставки (08:00 - 20:00) и продолжительностью окна времени доставки данному клиенту.

Например, окно времени доставки клиенту с 08:00 до 10:00, т.е. 2 часа. Длина рабочего дня с 08:00 до 20:00 - 12 часов. Разница 12 часов - 2 часов = 10 часов. Величина штрафа по такому заказу составит 10 часов * 10 минут = 100 минут штрафа.

Штраф другой город, мин. - увеличение рассчитанного штрафа на заданную величину для клиентов в другом городе.

Прогрессивный штраф применяется для того, чтобы отбрасывались в первую очередь заказы с наибольшим окном времени ("в течении дня") - их проще доставить вторым рейсом.
